# TechBase IT PowerShell Repository

Welcome to the official PowerShell repository for TechBase IT, a dedicated space for sharing and collaborating on PowerShell scripts tailored for all Microsoft Technology stacks. This repository serves as a central hub for IT professionals, system administrators, and developers seeking to automate and streamline their workflows with PowerShell.

## Contributors
This repository is actively contributed to by Darol Lucas and Dwayne Mc Gordon, who bring their extensive knowledge and experience in PowerShell scripting across the Microsoft Technologies stack. Their contributions form the backbone of this repository, ensuring that the scripts are not only functional but also adhere to best practices in PowerShell scripting.

## About Our Scripts
Our PowerShell scripts cover a wide range of Microsoft Technologies, aiming to provide solutions for automation, configuration, and management tasks. These scripts are designed to help TechBase IT and the broader community enhance efficiency, reduce manual errors, and improve overall system management.

### Areas Covered
- **Azure Cloud Services:** Automate deployment, management, and operations of Azure resources.
- **Active Directory:** Scripts for managing users, groups, and resources within Active Directory.
- **Windows Server Administration:** Automate tasks related to Active Directory, DNS, DHCP, and more.
- **Hyper-V:** Scripts to manage virtualized infrastructure.
- **Azure Stack HCI:** Automate ASHCI deployments.

## Getting Started
To get started with our PowerShell repository, clone this repository to your local machine or download the scripts directly from the web interface.

```powershell
git clone https://github.com/darollucas/powershell.git
```

Before running any script, ensure you read through the script's documentation and understand its purpose, parameters, and potential impact on your systems.

## Contribution Guidelines
We welcome contributions from the TechBase IT team and the broader community. If you have a script that you believe would benefit others in managing Microsoft Technologies, please follow our contribution guidelines outlined below:

1. **Fork the Repository:** Start by forking the repository to your GitHub account.
2. **Create a New Branch:** Make your changes in a new branch named after the feature or fix you're working on.
3. **Code Review:** Ensure your code adheres to PowerShell best practices and includes comments explaining the script's functionality.
4. **Submit a Pull Request:** Once you're ready, submit a pull request against our main branch. Include a detailed description of your changes and any relevant information.

## Reporting Issues
If you encounter any issues or have suggestions for improving the scripts, please use the GitHub Issues section of this repository. Provide as much detail as possible to help us understand and address the issue swiftly.
